# Web API 
Flask服务器实现了多个接口，本文档介绍了接口的使用方法、返回类型、参数说明等。

所有接口的请求体、响应体都采用`JSON(application/json; charset=utf-8)`格式。

在需要传递参数的接口中，应使用`POST`方法带请求体的方法，而不是使用URL参数。

接口应全部设置在`/api/`下，使用RPC风格的方式，小驼峰命名法命名。对于更复杂的接口，允许在接口名中包含斜杠。例如：`/api/login`、`/api/order/create`。


所有接口的响应体都包含以下字段：
- `status`：响应状态，整数。
当响应成功时，`status`为`0`；失败时，`status`为一个四位整数。其千位代表错误类型，详细的错误类型如下：

| 第一位数字 | 错误类型    | 说明            |
| ----- | ------- | ------------- |
| `1`   | 通用错误    | 请求错误          |
| `2`   | 授权/验证错误 | 用户登录状态、权限问题问题 |
| `3`   | 逻辑错误    | 各接口的具体错误      |
| `9`   | 系统错误    | 服务端错误         |

对于所有接口，都有以下通用的`status`状态码，部分`status`还有不同的HTTP状态码。
**通用错误：**

| 错误代码 | HTTP 状态码  | 说明  | 
| ------ | --- | --- |
| `1000`|  ？| 通用错误，其他未分类的错误。HTTP状态码非固定。 |
| `1001` | `400` | 请求参数错误，参数缺少或格式错误。 |
| `1002` | `405` | 请求方法错误 |
| `1003` | `404` | 请求资源不存在 |
| `1004` | `429` | 请求次数过多 |
| `2001` | `401` | 用户未登录 |
| `2002` | `403` | 用户权限不足 |
| `2003` | `401` | Token无效 |
| `2004`| `401` | Token过期 |
| `2005`| `401` | 旧设备登录 |
| `9001` | `500` | 服务器内部错误 |
| `9002` | `500` | 数据库错误 |
| `9003` | `500` | 文件系统错误 |
| `9004` | `500` | 其他系统错误 |

以下说明的所有接口的`status`仅包含此接口特有的`status`。

## 1. `/api/auth/login` 用户登录
- 请求方法：`POST`
- 请求参数：
    - `username`：用户名，字符串
    - `password`：密码，字符串。
- 响应参数：
    - `status`: 登录状态，整数。本接口特有的状态码如下：

| 状态码 | 说明 |
| ------ | --- |
| `3001` | 用户名或密码错误 |
| `3002` | 用户未启用 |
| `3003` | 重复登录 |
| `3004` | 设备重复登录 |

## 2. `/api/auth/logout` 用户退出登录
- 请求方法：`GET`
- 请求参数：无
- 响应参数：
    - `status`: 退出登录状态，整数。

## 3. `/api/auth/test` 测试登录
- 请求方法：`GET`
- 请求参数：无
- 响应参数：
    - `status`: 测试登录状态，整数


## 4. `/api/shop/dishes/getAll` 获取所有菜品
- 请求方法：`GET`
- 请求参数：无
- 响应参数：
    - `status`: 获取所有菜品状态，整数。
    - `data`: 所有菜品信息，字典。`data`中有以下字段：
        - `dishes`：菜品信息，字典。字典的key为分类名称，value为菜品数组。数组的每个元素为一个字典，包含菜品所有信息（信息与数据库记录的对应）。
        - `categories`：菜品分类，字典。提供了分类的id与名称的对应关系。


## 5. `/api/shop/dishes/get` 获取菜品信息
- 请求方法：`POST`
- 请求参数：
    - `id`：菜品id，整数。
- 相应参数：
    - `status`: 获取菜品信息状态，整数。
    - `data`: 菜品信息，字典。`data`字段的信息与数据库记录的对应。
    
| 状态码 | HTTP 状态码 | 说明 |
| ------ | ------ | --- | --- |
| `3001` | `400` | `id`参数缺少或格式错误 |
| `3002` | `404` | 菜品不存在 |

## 6. `/api/shop/dishes/update` 更新菜品信息
    
| 状态码 | HTTP 状态码 | 说明 |
| ------ | ------ | --- | --- |
| `3001` | `400` | 未更改任何信息 |
| `3002` | `404` | 菜品不存在 |

# Order Design
本文档介绍了本系统对于订单的设计，包括数据库、接口、逻辑等。

# 1. 数据库
存储订单信息的数据库位于`main.db`中的三张表，分别为：`orders`、`order_items`、`order_stats`。

## 1.1 `orders`表
`orders`表拥有以下字段：

1. `id`：表的主键，整数型，非空。用于唯一标识每个订单，`id`通常由14位整数构成，在创建新订单时，根据提供的参数自动生成。一个正确的`id`应包含以下内容：
- 时间：年、月、日。其中年占4位，月占2位，日占2位。
- 用户ID：用户ID占2位。来源于`users`表。
- 订单号：订单号占4位。每天从1开始递增。

例如：`20260701010001`代表在`2026`年`07`月`01`日时，用户`01`创建的第1个订单。

2. `creator`：创建订单的用户ID，整数型，来源于`users`表。非空。

3. `display_no`：可译为“流水号”，整数型，非空。与`id`不同，`display_no`用于给用户显示订单号；`id`用于在数据库作唯一标识。

4. `table_no`：订单所属的桌号，整数型，来源于`tables`表。非空。

5. `total_mount`：订单总金额，整数型，非空。

6. `note`：订单备注，文本型。

# Route Design
本文档介绍了在前端路由的设计（不包括`/api/`开头的由后端处理的路由）。

## 1. 核心思路
除了`/`、`/orders`、`/me`、`/develop`、`/serve`等三个顶级页面的路由，其他的路由都应为二级地址，这些路由的第一级应属于以下的分类：
 - `/me/`：用户相关路由。
 - `/shop/`： 店铺设置相关路由。
 - `/order/`： 订单单例的相关路由。



## 2. 顶级页面路由设计
1. `/`为首页，展示店铺的基本信息及快捷操作。

2. `/orders`为订单路由，展示所有订单及搜索订单功能。带`keyword`Query的为搜索页面。

3. `/me`为用户相关路由，展示用户的基本信息及操作。

4. `/`、`/orders`、`/me`均可通过三个页面下的导航栏切换。

5. `/develop`为开发选项路由，展示开发相关的功能。

6. `/develop`路由可通过`/`路由下的按钮切换。

7. `/serve`为出餐页面的路由，用户可以在该页面查看并确认出餐。

8. `/serve`路由可通过`/`路由下的按钮切换。
